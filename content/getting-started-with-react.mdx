# Getting Started with React

React is a powerful JavaScript library created by Facebook (now Meta) that has revolutionized the way we build user interfaces. It allows developers to create interactive, component-based web applications efficiently and effectively.

## What is React?

React is a declarative, efficient, and flexible JavaScript library for building user interfaces. It lets you compose complex UIs from small and isolated pieces of code called "components."

React's key features include:

- **Component-Based**: Build encapsulated components that manage their own state
- **Virtual DOM**: Efficient updates and rendering
- **One-Way Data Flow**: Makes it easier to reason about your application
- **JSX**: A syntax extension that makes writing components feel natural

## Setting Up Your First React Project

The easiest way to create a new React application is using Create React App or Next.js:

### Using Create React App

```bash
npx create-react-app my-app
cd my-app
npm start
```

### Using Next.js (Recommended)

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

## Understanding Components

Components are the building blocks of React applications. They let you split the UI into independent, reusable pieces.

### Function Components

The simplest way to define a component is using a JavaScript function:

```jsx
function Welcome() {
  return <h1>Hello, World!</h1>;
}
```

### Arrow Function Components

You can also use arrow functions:

```jsx
const Welcome = () => {
  return <h1>Hello, World!</h1>;
};
```

## Props

Props (short for properties) are how you pass data from parent to child components:

```jsx
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}

function App() {
  return (
    <div>
      <Greeting name="Rafael" />
      <Greeting name="World" />
    </div>
  );
}
```

## State Management

State allows components to create and manage their own data. Use the `useState` hook:

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}
```

## Event Handling

React events are handled using camelCase syntax:

```jsx
function Button() {
  const handleClick = () => {
    alert('Button clicked!');
  };

  return (
    <button onClick={handleClick}>
      Click Me
    </button>
  );
}
```

## Lists and Keys

When rendering lists, always include a `key` prop:

```jsx
function TodoList() {
  const todos = ['Learn React', 'Build an app', 'Deploy it'];

  return (
    <ul>
      {todos.map((todo, index) => (
        <li key={index}>{todo}</li>
      ))}
    </ul>
  );
}
```

## Conditional Rendering

You can conditionally render content using JavaScript operators:

```jsx
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? (
        <h1>Welcome back!</h1>
      ) : (
        <h1>Please log in.</h1>
      )}
    </div>
  );
}
```

## Best Practices

1. **Keep components small**: Each component should have a single responsibility
2. **Use descriptive names**: Component names should clearly indicate their purpose
3. **Lift state up**: Share state between components by moving it to a common ancestor
4. **Don't mutate state**: Always use the setter function provided by `useState`
5. **Extract reusable logic**: Create custom hooks for logic you want to reuse
